# Money Dates 💸

A personal finance course you *play*, not study — built for someone who finds money talk boring (and would rather be at barre).

The whole idea: **guess first, then find out.** Every week opens with a question, not a lecture. You have a guess, tap to reveal, and the lesson lands because you were curious first. A few weeks aren't reading at all — they're little interactive toys.

Designed to be done one cozy "money date" a week — twenty minutes, paired with something sweet.

## ✨ What makes it different

- **Hook-first cards.** Each card starts with a curiosity question. Perfect for doing together: one person reads the question, the other guesses, then you reveal the answer side by side.
- **Playable, not just readable.** Four weeks are hands-on:
  - 🧮 **Cost-per-use calculator** — punch in any purchase and see what it *really* costs per use
  - 📈 **Compound interest slider** — drag the numbers and watch your money grow (the "wow" moment)
  - 🛡️ **Spot the scam** — a quick legit-or-scam game
  - ⚖️ **Good debt / bad debt** — sort real scenarios and learn the difference
- **Written in real language** — skincare routines, Pilates, cost-per-wear, Malaysian banks and apps. No jargon.
- **Progress tracking** — mark each week done; it saves automatically.

## 🗺️ The journey — 27 weeks, 14 domains

**Mindset → Earn → Spend → Budget → Organize → Save → Protect → Borrow → Tax → Grow → Plan → Major Life Decisions → Partner & Family → Legacy**

Each step builds on the one before — money comes in, you learn to understand it, manage it, protect it, grow it, and finally plan a life and a partnership around it.

## 📖 How to use it

1. Open the site (or `index.html`)
2. Read the week's **question** — have a real guess
3. Tap **Reveal** (or **Try it** on interactive weeks)
4. Tap **Mark done** once you've had that week's money date
5. Swipe, use ← → keys, tap the dots, or use the domain chips to navigate

Do one a week. That's the whole pace.

## 🚀 Put it online for free

It's a single, dependency-free HTML file — ideal for GitHub Pages:

1. Create a new GitHub repository
2. Upload `index.html` and `README.md`
3. Go to **Settings → Pages**
4. Under "Branch", pick `main` and `/ (root)`, then **Save**
5. Live at `https://<your-username>.github.io/<repo-name>/` in a couple of minutes

No build step, no frameworks, no setup.
